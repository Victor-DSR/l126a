- Faça e teste cada função isoladamente. Um programa como o abaixo poderia ser usado para testar a função de potência:
   ```c
   #include <stdio.h>

   double potencia(double x, int n)

     //…
   }

   void testa_potencia()
   {
     printf("Teste da potência\n");
     printf("%lf = %lf\n", 25.0, potencia(5.0, 2));
     printf("%lf = %lf\n", 6.25, potencia(2.5, 2));

   ```
- Você pode desenvolver outras função auxiliares. Por exemplo, uma função que calcula o valor absoluto poderia simplificar o teste do valor absoluto no algoritmo de Heron.
- Uma forma de elaborar um algoritmo para solucionar o problema a ser resolvido por essas funções é tentar com um exemplo.
   A função para calcular a potência, por exemplo. O resultado é obtido à partir de multiplicações, e o número de multiplicações não é sempre o mesmo, depende do expoente. Em casos assim, um laço de repetição é comum para resolver. Nesse caso, dá para definir o número de repetições à partir dos argumentos da função, então dá para partir dessa informação para montar o laço.

   No laço serão realizadas as multiplicações. As vezes simular com um exemplo real facilita definir os comandos à repetir. Digamos que queiramos calcular `10⁴`. O resultado é `10*10*10*10` que é 10000. Como poderíamos obter esse resultado uma multiplicação por vez? Uma forma seria:
   ```
      10*10 -> 100
      100*10 -> 1000
      1000*10 -> 10000
   ```
   São três multiplicações, e se fosse `10⁵` seriam 4, parece que o número de multiplicações é um a menos que o expoente — fácil de armar o laço. 
   Que valores são multiplicados? Na primeira linha, são duas vezes o valor recebido, mas nas demais não.
   Vamos precisar de duas variáveis para implementar essa multiplicação:
   ```
      a * b
      a * b
      a * b
   ```
   O primeiro valor multiplicado muda entre uma multiplicação e outra, e parece ser o resultado da multiplicação anterior.
   ```
      a = a * b
      a = a * b
      a = a * b
   ```

   Na primeira repetição não temos o resultado da multiplicação anterior, mas a variável que vai conter esse valor pode ser inicializada para que tenha o valor que necessitamos.
   ```
      a = 10
      b = 10
      a = a * b
      a = a * b
      a = a * b
      a tem o resultado
   ```
   Agora temos linhas iguais para implementar o laço, e outras linhas, que devem estar fora do laço.

   Uma outra forma de se realizar as multiplicações, com o mesmo resultado, seria:
   ```
      1*10 -> 10
      10*10 -> 100
      100*10 -> 1000
      1000*10 -> 10000
   ```
   Nessa forma, temos o número de repetições igual ao expoente. Essa forma tem a vantagem de, se nenhuma repetição for feita, a variável que tem o resultado já tem o resultado correto para o caso de o expoente ser 0, e economizamos um `if`.
   No caso anterior, se o número de repetições for 0, o que vai acontecer com o expoente 1, a variável tem o valor correto?
